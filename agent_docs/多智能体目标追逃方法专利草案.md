4. 动作空间

环境中智能体（追捕无人机与任务目标）遵循离散时间下的动力学约束：
$$ p_hi(s) = p_hi(s-1) + v_hi(s) * dt $$
$$ v_hi(s) = max(v_hi(s-1) + a_hi(s) * dt, v_h^max)$$

其中p_hi表示第s时刻智能体的位置，v_hi表示第s时刻智能体i的速度，a_hi表示第s时刻智能体的加速度；此外，追捕无人机和任务目标均有其对应的最大运动速度v_h_max 和 v_t_max, 以及最大加速度。

# 说明书

## 技术领域

本发明涉及多智能体强化学习与多无人机协同控制技术领域，具体涉及一种基于多智能体强化学习的目标追逃方法。

## 背景技术

多无人机目标追逃任务是典型的多智能体对抗协作问题。现有技术中，常见方法存在如下不足：
1. 训练任务分布单一，Hunter数量固定、Target行为固定，导致策略对场景变化泛化不足；
2. 评估阶段缺乏统一任务基准，不同实验设置难以直接横向比较；
3. 奖励函数多以距离驱动为主，Hunter易聚团追击而非形成有效包围，目标仍可由几何缺口逃逸。

因此，需要一种同时具备场景泛化能力、可复现评估能力与围捕协同能力的目标追逃方法。

## 发明内容

### 发明目的

本发明的目的在于提供一种基于多智能体强化学习的目标追逃方法，以解决现有技术中泛化性不足、评估不可比和围捕协同弱的问题。

### 技术方案

为实现上述目的，本发明采用如下技术方案：

S1. 追逃任务建模：
构建多Hunter‑单Target的连续控制环境，定义状态转移、局部观测、动作约束与终止条件；局部观测由自身信息、邻居信息、目标信息和共享记忆组成。

S2. 训练域随机化：
在训练过程中随机采样Hunter数量与Target行动策略，形成可变规模、可变对手行为的训练任务分布；通过角色共享策略训练Hunter，使策略在不同任务规模间迁移。

S3. 固定评估场景数据集：
评估前预构建场景数据集，每个任务样本包含场景大小、Hunter数量、Target策略和随机种子；评估阶段按任务样本逐一恢复执行，保证评估结果跨实验可复现、可比较。

S4. 围捕几何奖励：
在复合奖励中引入escape_gap reward，通过计算Target周向最大潜在逃脱夹角来刻画围捕缺口；将该几何量转化为奖励信号，驱动Hunter形成环绕封堵策略，而不仅是单纯最近距离追击。

S5. 策略输出与执行：
将当前观测输入训练后的策略网络，输出Hunter动作并推进环境状态更新，实现目标追逃决策。

### 有益效果

相较于现有技术，本发明具有以下有益效果：
1. 通过训练期Hunter数量与Target策略随机变化，提高策略对多场景任务的泛化能力；
2. 通过固定评估场景数据集与恢复式评估机制，提高评估稳定性并支持跨实验公平对比；
3. 通过escape_gap围捕几何奖励，显式鼓励Hunter形成包围圈，提升围捕效率与协同性。

## 具体实施方式

以下结合具体实施例对本发明作进一步说明。

### 实施例一：追逃环境建模

在二维连续空间中设置多个Hunter与一个Target。每个智能体动作为二维连续向量，采用速度控制或加速度控制进行状态更新。Hunter侧观测由五类特征拼接：
1. own_obs：自身归一化位置与速度；
2. neighbor_obs：最近同阵营邻居相对特征；
3. target_obs：目标相对位置、速度与可见位；
4. memory_obs：目标不可见时的共享记忆；
5. coord_summary_obs：可选协同摘要特征。

### 实施例二：训练域随机化（泛化能力）

训练阶段每隔预设episode间隔或按预设概率进行任务重生成。重生成时随机采样：
1. Hunter数量：从候选集合中采样；
2. Target策略：从learn、random、patrol中采样；
3. 随机种子：从给定seed区间采样。

该机制使策略在不同团队规模、不同对手行为模式和不同初始扰动下训练，提升场景泛化能力。

### 实施例三：固定评估场景数据集（跨实验可比）

评估前离线生成固定任务集，单个任务记录：
(world_size, hunters_num, target_policy, seed)。

评估阶段读取该任务集并采用recover模式reset，使每个任务在不同实验中保持一致初始化与随机过程。最终统计总体指标与按Hunter数量分桶指标，确保跨实验可比。

### 实施例四：escape_gap reward（包围式追捕）

在每个时间步，对Target周边有效Hunter进行几何分析：
1. 选取escape_radius内的有效Hunter；
2. 计算每个Hunter对应的角度阻塞区间；
3. 在离散角度bins上累积阻塞掩码；
4. 计算最大未阻塞扇区（最大潜在逃脱夹角）；
5. 基于Target运动方向与逃脱缺口中心方向关系计算方向得分；
6. 将方向得分映射为Hunter侧与Target侧escape_gap奖励。

该奖励促使Hunter主动压缩逃脱缺口并形成包围圈，实现由“追击”向“围捕”的策略转变。

### 实施例五：复合奖励与训练执行

总奖励由如下项线性组合：
R = R_hunter_base + R_target_base + R_streak + R_capture + R_collision + R_speed + R_escape_gap。

其中：
1. 基础奖励驱动接近目标；
2. streak奖励鼓励持续压制；
3. capture奖励用于捕获事件强化；
4. collision与speed项抑制危险行为和速度振荡；
5. escape_gap项提升围捕几何质量。

训练执行层采用角色共享Runner进行采样、更新、日志与评估，周期性输出评估结果并保存最优模型。
