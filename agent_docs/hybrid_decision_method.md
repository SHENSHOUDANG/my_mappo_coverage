我现在要基于 无人机集群的区域搜索与目标追捕任务 的一种混合决策方法申请一份专利，其将整个任务划分为了 区域搜索——任务目标分配——目标追捕 三个大致阶段。

区域搜索和任务目标分配均采用传统的基于建模与联合优化的方法求解，有很强的可解释性，并且处于最终任务执行前的阶段，有相对充裕的时间供使用者进行决策和方案调整。

而目标追捕阶段则使用基于MARL的方法，通过事先基于 N v 1 目标追捕任务训练好的追捕无人机策略模型来执行目标追捕子任务。该方法的具体细节可以参考上传的附件，该附件即为该MARL方法对应的专利申请书，在编写本混合决策方法的专利申请书时，需要参照该附件的格式进行编写。

# 多无人机混合决策方法与系统

# 研究背景
当前的许多无人机集群应用中的群体决策过程都可以抽象为“搜索——分配——执行”三个阶段的任务流程：
* 搜索阶段: 对指定任务区域开展覆盖式搜索，在指定时间内尽可能多地找到任务目标；
* 分配阶段: 每当发现新的任务目标 / 完成一个区域搜索后，对目前发现的任务目标进行任务分配，派遣指定数量无人机处理任务目标；
* 执行阶段: 根据任务分配结果，组成任务小组执行具体任务。

目前绝大多数无人机集群任务还是采用基于传统规则优化的方式进行群体任务决策，通过一系列规则和专业知识对整个问题进行建模，并通过优化求解等方式计算得到最终的规划决策结果，此类方法有很强的可解释性，且规划结果相对稳定，但是存在几个缺陷：
 - 决策问题建模难度大，且每个任务都需要有较强专业背景的技术人员进行问题建模。许多群体决策任务难以明确定义“最优”的决策方案；
 - 随着集群规模的增长，问题复杂度急剧增加，需要增加大量显式约束，难以进行有效扩展和维护；
 - 只能应对建模范围内的场景，对于快速动态变化的场景难以适应；
 - 规划求解时间不稳定，往往受任务复杂程度影响

 相比之下，现在基于强化学习的多智能体决策方法开始逐步走进大家的视野，其通过多智能体在任务环境中的反复探索与环境反馈自发进行策略学习，无需对具体任务决策问题进行显式建模，因此得到了大量关注，并且相比于传统方法，基于强化学习的方法有非常独特的优势：
 - 规模扩展难度低，对于采用CTDE训练方式的多智能体系统而言，其天然支持智能体数量的弹性扩展，无需对系统进行修改；
 - 无需对决策问题显式建模，模型通过反复探索尝试学习到“最优”策略；
 - 有较强的泛化能力（因为在环境探索过程中可以遭遇各种可能的case）；
 - 规划求解时间稳定，每次决策都只是一次模型前向推理过程；

 但是这并不意味着强化学习方法就足以替代传统的规划方法，因为强化学习方法还存在一个巨大的缺陷：缺乏可解释性，这对于实际使用者来说难以对其最终决策结果有准确的心理预期，因此存在一定的安全风险隐患，也难以直接根据实际需要对决策结果进行人工干预或定向调整（决策倾向调整）。  这使得该方法几乎只能用于人在环中的决策系统，并且人工干预的方式还相对受限。
 
 因此在实际应用中，往往不会在整个任务过程都采用纯MARL的决策方法，而是保留一部分的传统规划方法进行决策。为此，本文提出了一个结合传统优化决策与强化学习决策的混合协同决策方法，一方面通过显式的角色分工明确集群中各个无人机的能力边界，仅对面临高动态博弈的复杂子任务采用基于MARL的决策方法以获取更好的执行结果。


# 任务定义
无人机集群组成包括多个Hunters和Explorers无人机，而任务环境中还有多个位置/运动速度未知的任务目标Targets：
 - Hunters进行目标围捕
 - explorer进行目标搜索与跟踪；
 - Target为被追捕目标，会沿着固定路线运动，或主动躲避靠近的Hunters和Explorers无人机

任务流程：
1. 在任务面板上新建任务，指定world_size大小、hunters数量N、explorers数量M、以及任务目标数量K，以及任务总时间T（单位为秒，环境每次step的时间间隔dt=0.1秒）。

每个explorer都有其对应的感知范围r_perc、最大速度限制
每个hunter都有对应的最大速度限制

此外每个explorer和hunter都有自己对应的总续航（里程数）限制，每次环境step会消耗 velo * dt + c 的里程，c是定值，确保无人机悬停待命的情况下也消耗续航

随机设置任务目标K的位置，以及行动策略（random、patrol、escape）

2. 航线预规划（需要人工确认，对重叠率参数、hunters待命模式进行设置，再触发出发预规划，对于预规划结果不满意的情况，可以再重新调整参数规划）：
（1）规划1个explorer对该区域的覆盖式搜索航线（弓字形航线，弓字航线的长边与任务区域的横坐标平行），随后等分为M个子航线，由各个explorer进行执行，往返运动进行区域搜索。该搜索航线应当与地图边界保持一定距离，同时弓字形的间距则基于explorer的感知范围和重叠率`overlap_rate`设定（在弓字形航线的相邻两条平行航线间距，应该等于 2*r_perc - r_perc * overlap_rate，即有`overlap_rate`比例的explorer感知区域会发生重叠，从而确保有效的覆盖式搜索）；

（2）根据hunters的待命模式分为两种规划方式：
（2-1）split：生成N条与任务区域纵坐标平行的N个hunters的预设航线（航线间距相等，hunters在航线上往返运动等待被分配任务目标）；
（2-2）zone：hunters还有一种特殊航线模式，便是根据explorer数量平分成M个hunters小组，每个组成一个方阵（按照hunters小组中hunters数量，计算最接近且大于等于该数的平方数来排列方阵位置，并随机分配hunters在方阵中的位置，其实际实现与训练参数 `hunters_in_zone`为True时的处理一致），始终跟随指定explorer运动。

3. 初始化一个任务目标池（维护当前已发现的任务目标），和一个无人机资源池（当前子区域内的hunters和explorers的位置、速度和任务状态（是否执行任务，对应任务目标ID））

4. 每个explorer和hunter都出现在其对应预设航线的起点位置，explorer按照预设的最大运动速度运动

hunters为split待命模式时，其运动速度会在[0, max_velo]中随机采样，实现类似hunters在环境中随机分布的效果
hunters为zone待命模式时，则按照（2-2）的方式出现在explorer当前运动速度方向后方

5. 执行预规划航线（手动确认），此时环境中的所有智能体开始随着环境的step进行运动。
运动过程中，只有explorer可以时刻获取和更新感知范围内的Target目标当前信息（位置、速度以及ID），并将发现的目标信息共享到一个全局的任务目标信息池中。每当任务目标信息池出现新任务目标时，触发任务分配；

6. 任务分配执行：基于匈牙利算法实现：
（1）根据目标value决定至少需要多少hunters进行处理，每个目标至少需要1个hunters处理（至多5个），同时也需要一个explorer；
（2）根据hunter/explorer与Target距离、目标价值、hunters当前已分配任务目标、剩余续航等计算匹配成本（要求尽可能倾向于选择高价值、近距离目标，且避免频繁切换任务目标，即优先分配给当前无任务目标的hunter进行处理）；   对于距离大于设定阈值的hunter/explorer不分配任务目标
（3）通过匈牙利算法求解每个子区域内的任务分配结果；

匹配成本计算过程中涉及多个子项，每个子项也有对应的权重因子。使用者可以通过人工调整因子大小来修改任务分配的决策倾向，也可以将配置好的决策倾向保存命名，后续可以直接加载使用

7. 对于每一组通过任务分配确定的hunters、explorer和target，先修改其在任务目标池和无人机资源池中的状态。等到explorer自己成功感知到target后，正式开始一个独立目标追捕子任务。

- 被分配到任务目标的explorer从SEARCH状态切换到Track状态，暂时中断区域搜索航线的执行，始终朝向被分配到的Target目标位置飞行（该过程不受任何hunters的状态限制，一直持续到本次追捕子任务结束为止）
- 被分配到任务目标的hunters状态换变更为追捕状态，但是暂时不离开自身的预设航线，直至追捕子任务开始，才根据实际的策略进行追捕动作

- 追捕子任务会创建一个UAVPursuitEnv，根据当前hunters和target位置、速度等状态初始化该环境，并进行step，将step结果更新到总任务环境中）

8. 当围捕子任务环境到达终止状态后，更新任务目标池信息，将任务目标从任务目标池中删除

 - explorer重新切换到Search状态，返回其对应搜索航线上最近的航点重新开始搜索
 - hunters且重新返回其对应预设航线中断位置    （该过程不考虑碰撞）
 同时也更新任务目标池和无人机资源池信息。

 无人机资源池信息更新时（表示空闲无人机资源增加后）触发任务分配，回到步骤6.


该流程中可能还存在几个特殊情况，其大致流程如下：
A1. explorer A在被分配到新的target A，处于Track状态时，发现了另一个Target B，此时将触发任务目标分配，理想情况下会有另一个explorer B负责处理该任务目标（只要当前空闲的explorer数量>0），若此时当前子区域内没有空闲explorer，则任务分配失败，直到步骤8有新的空闲无人机资源释放时，重新进行任务分配
A2. 此时由于新发现的Target B是由explorer A进行更新，且很可能不处于explorer B的感知范围内，因此可能出现explorer B接近Target B的过程中，Target B运动到了所有explorer的感知范围以外，此时视作Target B的追捕子任务未成功开始，令explorer B重新返回上次中断的搜索航线航点（此时explorer也视作空闲状态），hunters也重新标记为空闲状态；

将Target B移除任务目标池，并更新无人机资源池
