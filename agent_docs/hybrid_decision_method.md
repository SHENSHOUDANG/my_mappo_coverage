# 多无人机混合决策方法与系统

# 研究背景
当前的许多无人机集群应用中的群体决策过程都可以抽象为“搜索——分配——执行”三个阶段的任务流程：
* 搜索阶段: 对指定任务区域开展覆盖式搜索，在指定时间内尽可能多地找到任务目标；
* 分配阶段: 每当发现新的任务目标 / 完成一个区域搜索后，对目前发现的任务目标进行任务分配，派遣指定数量无人机处理任务目标；
* 执行阶段: 根据任务分配结果，组成任务小组执行具体任务。

目前绝大多数无人机集群任务还是采用基于传统规则优化的方式进行群体任务决策，通过一系列规则和专业知识对整个问题进行建模，并通过优化求解等方式计算得到最终的规划决策结果，此类方法有很强的可解释性，且规划结果相对稳定，但是存在几个缺陷：
 - 决策问题建模难度大，且每个任务都需要有较强专业背景的技术人员进行问题建模。许多群体决策任务难以明确定义“最优”的决策方案；
 - 随着集群规模的增长，问题复杂度急剧增加，需要增加大量显式约束，难以进行有效扩展和维护；
 - 只能应对建模范围内的场景，对于快速动态变化的场景难以适应；
 - 规划求解时间不稳定，往往受任务复杂程度影响

 相比之下，现在基于强化学习的多智能体决策方法开始逐步走进大家的视野，其通过多智能体在任务环境中的反复探索与环境反馈自发进行策略学习，无需对具体任务决策问题进行显式建模，因此得到了大量关注，并且相比于传统方法，基于强化学习的方法有非常独特的优势：
 - 规模扩展难度低，对于采用CTDE训练方式的多智能体系统而言，其天然支持智能体数量的弹性扩展，无需对系统进行修改；
 - 无需对决策问题显式建模，模型通过反复探索尝试学习到“最优”策略；
 - 有较强的泛化能力（因为在环境探索过程中可以遭遇各种可能的case）；
 - 规划求解时间稳定，每次决策都只是一次模型前向推理过程；

 但是这并不意味着强化学习方法就足以替代传统的规划方法，因为强化学习方法还存在一个巨大的缺陷：缺乏可解释性，这对于实际使用者来说难以对其最终决策结果有准确的心理预期，因此存在一定的安全风险隐患，也难以直接根据实际需要对决策结果进行人工干预或定向调整（决策倾向调整）。
 
 在许多实际应用场景中，反而是必须要使用基于传统的规划方法。为此，本文提出了一个结合传统优化决策与强化学习决策的混合协同决策方法，一方面通过显式的角色分工明确集群中各个无人机的能力边界，同时


# 任务定义
无人机集群组成包括多个Hunters和Explorers无人机，而任务环境中还有多个位置/运动速度未知的任务目标Targets：
 - Hunters进行目标围捕
 - explorer进行目标搜索与跟踪；
 - Target为被追捕目标，会沿着固定路线运动，或主动躲避靠近的Hunters和Explorers无人机

任务流程：
1. 环境初始时刻指定环境的world_size，以及hunters、explorers、Targets的数量（N，M，K）和对应运动性能（最大速度、加速度等）与任务时间要求T
2. 对任务区域进行分组，划分为S个等面积的子区域 ，其中子区域划分数量根据explorer的最大飞行速度与任务时间要求T进行大致估算：（1）估算单个explorer在T任务时间内的感知范围覆盖面积（除去重叠区）；（2）要求每个子区域分配的Explorer数量，可以在最多T/2时间内完成1次子区域的覆盖式搜索；（3）根据区域分组结果，将hunters也等分为S份分配给对应的子区域；
3. 进行区域分组后，每个子区域都会得到n个hunters和m个explorers（除了最后一个子区域，可能因为无法等分存在差异），都进行航线预规划：（1）规划1个explorer对该区域的覆盖式搜索航线（弓字形航线），随后等分为m个子航线，由各个explorer进行执行，往返运动进行区域搜索；（2）根据子区域的长边等分得到n个hunters的预设航线（航线与子区域的短边平行，间距相等，往返运动（不需要规划为弓字形）
4. 每个explorer和hunter都出现在其对应预设航线的起点位置，随后按照预设的最大运动速度运动；
5. 运动过程中，只有explorer可以时刻获取到感知范围内的Target目标当前信息，并触发任务分配；
6. 任务分配执行：基于匈牙利算法实现：（1）根据目标value决定至少需要多少hunters进行处理，每个目标至少需要1个hunters处理，同时也需要一个explorer；（2）根据hunter/explorer与Target距离、目标价值、hunters当前任务目标计算匹配成本（要求尽可能倾向于选择高价值、近距离目标，且避免频繁切换任务目标，即优先分配给当前无任务目标的hunter进行处理）；（3）通过匈牙利算法求解每个子区域内的任务分配结果；
7. 对于每一组通过任务分配确定的hunters、explorer和target，形成一个独立目标围捕子任务，此时explorer从SEARCH状态切换到Track状态，始终朝向Target飞行，同时创建一个对应的UAVPursuitEnv，根据当前hunters和target状态初始化该环境，并进行step，将step结果重新设置到总任务环境中）；
8. 当围捕子任务环境到达终止状态后，explorer重新切换到Search状态，返回其对应搜索航线上最近的航点重新开始搜索，hunters且重新回到其对应预设航线
